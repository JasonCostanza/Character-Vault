# Weapon Traits & Enhancements

Reference for the Weapons module's two distinct item-tagging systems: **Traits** (system-defined chips) and **Enhancements** (user-created catalog).

---

## Two Separate Systems

| | Traits | Enhancements |
|---|---|---|
| **Source** | Built-in system catalogs per game system | User-created, stored on the module |
| **Scope** | Per weapon | Module-level catalog → attach to weapons |
| **Stored on** | `weapon.traits[]` | `data.content.enhancementCatalog[]` + `weapon.attachedEnhancements[]` |
| **Purpose** | Descriptive tags (Finesse, Two-Handed, etc.) | Numeric bonuses (attack, pool, damage dice) |
| **Game systems** | D&D 5e, PF2e, Daggerheart | PF2e (striking), SR6 (pool), CpRed (attack) |

---

## Traits

Traits are descriptive labels that characterize a weapon (e.g., Finesse, Versatile, Reach). They display as chips in the weapon card and detail modal.

### System Trait Catalogs

`getSystemTraitCatalog()` returns the active system's built-in trait list:

| System | Catalog variable | Key prefix |
|---|---|---|
| D&D 5e | `WEAPON_TRAITS_DND5E` | `dnd5e.` |
| `custom` | `WEAPON_TRAITS_DND5E` | `dnd5e.` (same as 5e) |
| PF2e | `WEAPON_TRAITS_PF2E` | `pf2e.` |
| Daggerheart | `WEAPON_TRAITS_DAGGERHEART` | `daggerheart.` |
| VtM, SR6, CpRed, etc. | `[]` (empty) | — |

### Trait Data Structure

Each entry in a weapon's `traits` array:

```js
{
  key: 'dnd5e.finesse',   // system-prefixed identifier
  value: null              // numeric value (non-null only for traits like Versatile: '1d8')
}
```

### Custom Traits

Users can also create custom traits not in any catalog. Custom traits are stored in `data.content.customWeaponTraits[]`:

```js
{
  key: 'custom.wt_abc123',
  name: 'Silvered',
  description: 'Counts as silvered for bypassing resistance.'
}
```

Custom traits use the `'custom.'` key prefix. They share the same `traits[]` array on the weapon as system traits.

### Trait Resolution

`resolveWeaponTrait(traitEntry, content)` converts a `{ key, value }` pair into a displayable object:

```js
resolveWeaponTrait({ key: 'pf2e.agile', value: null }, content)
// → { key: 'pf2e.agile', name: 'Agile', description: '...', takesValue: false, isCustom: false }

resolveWeaponTrait({ key: 'custom.wt_abc123', value: null }, content)
// → { key: 'custom.wt_abc123', name: 'Silvered', description: '...', takesValue: false, isCustom: true }
```

### Trait Normalization

`normalizeWeaponTraits(traits, content)` handles migrating old string-based trait arrays to the keyed format:

```js
normalizeWeaponTraits(['Finesse', 'dnd5e.light'], content)
// → [{ key: 'dnd5e.finesse', value: null }, { key: 'dnd5e.light', value: null }]
```

String inputs are matched by normalized name against the current system catalog. Unmatched strings become custom traits via `findOrCreateCustomTrait()`.

## Enhancements (Phase 3)

Enhancements are user-created upgrades that live in a module-level catalog and attach to individual weapons. They provide numeric bonuses that feed directly into roll calculations.

### Data Structures

**Module-level catalog** (`data.content.enhancementCatalog[]`):

```js
{
  key: 'enh_abc123',      // unique, generated by weaponsGenerateEnhancementKey()
  system: 'pf2e',          // game system this enhancement is for
  name: '+1 Striking',     // display name
  description: 'Adds a die to damage rolls.', // optional flavor text
  damageDiceBonus: 1,      // PF2e only: extra damage dice (striking bonus)
  poolBonus: 2,            // SR6 only: adds to dice pool
  attackBonus: 1           // CpRed only: adds to attack roll
}
```

**Per-weapon attachment** (`weapon.attachedEnhancements[]`):

```js
['enh_abc123', 'enh_def456']   // array of enhancement keys from the catalog
```

### Enhancement Lifecycle

```
1. User opens Catalog Editor (openEnhancementCatalogModal)
   ↓
2. Creates a new enhancement (openEnhancementInlineForm → draft)
   ↓
3. Draft saved to data.content.enhancementCatalog[]
   ↓
4. User opens Weapon Edit Modal → Enhancements section
   ↓
5. Clicks "Attach" → openEnhancementPickerPanel
   ↓
6. Picks from available (unattached) enhancements
   ↓
7. Enhancement key pushed to weapon.attachedEnhancements[]
   ↓
8. Roll calculations include enhancement bonus
   ↓
9. "Detach" removes the key from weapon.attachedEnhancements[]
   (enhancement stays in catalog for re-use)
```

### Computed Bonus Functions

These are called during roll dispatch to include enhancement bonuses:

| Function | System | Bonus Type | How it works |
|---|---|---|---|
| `weaponsComputeEnhancementAttackBonus(weapon, content)` | CpRed | Attack roll bonus | Sums `e.attackBonus` on attached enhancements where `e.system === 'cpred'` |
| `weaponsComputeEnhancementPoolBonus(weapon, content)` | SR6 | Pool size bonus | Sums `e.poolBonus` on attached enhancements where `e.system === 'sr6'` |
| `weaponsComputeEnhancementStrikingBonus(weapon, content)` | PF2e | Damage dice count | Sums `e.damageDiceBonus` on attached enhancements where `e.system === 'pf2e'` |
| `weaponsApplyStrikingBonus(diceStr, bonus)` | PF2e | Applies to dice notation | Increments die count: `'1d8'` + bonus `1` → `'2d8'` |

### Pool Auto-Compute

For pool-based systems (VtM, SR6), weapons can compute their pool size dynamically from live stat values:

- `weapon.poolAutoCompute: false` — use `weapon.poolSize` directly
- `weapon.poolAutoCompute: true` — compute from `weapon.poolAttribute` + `weapon.poolSkill` stats, plus `weapon.poolAdjustment`, plus enhancement pool bonus

`weaponsComputeEffectivePool(weapon, content)` handles both paths:

```js
// Manual pool
weaponsComputeEffectivePool({ poolAutoCompute: false, poolSize: 5 }, content)
// → 5

// Auto-computed
weaponsComputeEffectivePool({ poolAutoCompute: true, poolAttribute: 'Agility', poolSkill: 'Firearms', poolAdjustment: 1 }, content)
// → getStatValue('Agility') + getStatValue('Firearms') + 1 + enhancementPoolBonus
```

## Attack Archetypes

`getAttackArchetype(gameSystem)` determines how attacks are rolled:

| Archetype | Systems | Behavior |
|---|---|---|
| `'A'` | D&D 5e, PF2e, CpRed, Mothership | d20 + modifier attack, damage dice separately |
| `'B'` | Custom, CoC, Daggerheart | Fixed modifier roll (no separate attack die) |
| `'C'` | VtM, SR6 | Pool roll (NdX, count successes) |

Roll dispatch in `renderPlayBody()` checks the archetype to build the correct `putDiceInTray` call.

**Attack bonus override**: If `weapon.attackBonusOverride !== null`, that value is used instead of the computed bonus (bypasses proficiency + ability modifier). Displayed with a visual indicator on the weapon card.

## Window Exports

All pure functions from the weapons module are exposed on `window` for tests and cross-module use:

```js
window.weaponsGenerateEnhancementKey(catalog)    // generates unique 'enh_' key
window.weaponsFindEnhancement(content, key)       // lookup by key in catalog
window.weaponsGetAttachedEnhancements(weapon, content)  // resolved array of attached enhancements
window.weaponsGetAvailableEnhancements(content, weapons, system)  // catalog entries not attached to any weapon
window.weaponsApplyStrikingBonus(diceStr, bonus)  // applies striking bonus to dice notation
window.weaponsComputeEnhancementPoolBonus(weapon, content)
window.weaponsComputeEnhancementAttackBonus(weapon, content)
window.weaponsComputeEffectivePool(weapon, content)
window.getSystemTraitCatalog()                    // returns active system's trait list
window.resolveWeaponTrait(traitEntry, content)    // converts key to displayable object
window.normalizeWeaponTraits(traits, content)     // migrates string arrays to keyed format
window.weaponsComputeAttackBonus(weapon)          // total attack bonus (ability + proficiency + override)
window.weaponsFormatDamageSummary(weapon, content)  // formatted damage string for weapon card
window.getAttackArchetype(sys)                    // returns 'A', 'B', or 'C'
```

## Adding Enhancement Fields for a New System

To add enhancement support for a new game system (e.g., a new bonus type):

1. **Add a field to the enhancement data shape** in `openEnhancementInlineForm()` — add an input for the new field and wire it to `draft.newField`.
2. **Add a compute function** — `weaponsComputeEnhancementNewBonus(weapon, content)` that sums `e.newField` for matching system entries.
3. **Use it in roll dispatch** — in `openWeaponActionModal()`, call the compute function and include it in the roll expression.
4. **Export on `window`** at the bottom of the IIFE.
5. **Add tests** to `tests/module-weapons-enhancements.test.js`.
